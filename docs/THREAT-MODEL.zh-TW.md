# 威脅模型

[English](THREAT-MODEL.md) | 繁體中文

## 安全目標

當其中一層儲存設備故障或受損時，`mirror-backup` 的目標是保留所選 Windows 資料的機密、版本化且可還原副本。

本設計假設：

- 備份執行期間，電腦本身可信任；
- Windows 帳戶與系統管理員安全邊界尚未遭入侵；
- restic 的加密與版本庫格式符合其公開文件；
- 至少有一組版本庫密碼記錄在電腦以外。

## 防護方式

| 威脅 | 防護方式 |
|---|---|
| 誤刪或不希望保留的修改 | 具有保留政策的版本化快照 |
| 線上版本庫遺失 | 完全獨立的卸除式版本庫 |
| 勒索軟體存取線上儲存 | 實際拔除的離線磁碟 |
| 接錯卸除式磁碟 | 比對本機與磁碟媒體標記，再驗證版本庫 ID |
| 版本庫靜默損壞 | 輪替執行 `restic check --read-data-subset` |
| 備份完成但無法還原 | 精確路徑 canary 還原、`--verify` 與 SHA-256 比對 |
| 更換整台電腦 | 密碼管理器或密封紙本密碼，加上可攜式 restic 救援工具包 |
| 密碼出現在命令歷程 | 透過暫時的程序環境將密碼交給 restic，不放在命令列參數 |

## 明確不處理的範圍

- 防止已經以同一使用者或系統管理員身分執行的程序取得版本庫密碼。
- 保護仍連接在電腦上的卸除式磁碟。
- 取代端點防護、作業系統更新或 NAS 存取控制。
- 保證線上版本庫或網路永遠可用。
- 自動安全備份所有應用程式資料庫；部分應用程式需要自己的匯出或暫停寫入機制。

## 重要操作規則

1. 線上與離線版本庫使用不同的隨機密碼。
2. 在正式依賴系統前，先把兩組密碼記錄在電腦以外。
3. 離線更新驗證成功後，立即安全退出並拔除磁碟。
4. 使用外部密碼紀錄測試救援，不可只測試 DPAPI。
5. Windows 帳戶密碼變更後，重新安裝定期工作。
6. 即使 restic 已建立快照，只要 canary 還原失敗，就視為備份失敗。

## 剩餘風險

- 惡意系統管理員可以修改已安裝的指令稿、定期工作、DPAPI 密碼或備份來源。
- 複製媒體標記不等同硬體認證；版本庫 ID 與密碼驗證仍負責避免誤用無關版本庫。
- 來源檔案可能包含秘密。加密只保護靜態版本庫，不保護已解鎖的 Windows 工作階段。
- 保留與清理作業會刻意刪除已無快照引用的版本庫資料；預設流程只對線上版本庫執行這些作業。
